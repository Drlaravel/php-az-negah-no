### فصل ششم: برنامه‌نویسی شیءگرا (OOP)

## 6.1 معرفی برنامه‌نویسی شیءگرا

برنامه‌نویسی شیءگرا (OOP) یک پارادایم برنامه‌نویسی است که بر اساس مفاهیم "شیء" و "کلاس" بنا شده است. این روش به شما امکان می‌دهد تا برنامه‌های خود را به صورت منظم‌تر و قابل‌توسعه‌تر بنویسید. در OOP، داده‌ها و رفتارهای مرتبط با آن‌ها در قالب اشیاء تعریف می‌شوند.

### 6.1.1 کلاس‌ها و اشیاء

یک **کلاس** یک الگو یا قالب برای ایجاد اشیاء است. یک **شیء** یک نمونه از یک کلاس است که ویژگی‌ها و رفتارهای کلاس را به ارث می‌برد. برای تعریف یک کلاس در PHP از کلمه کلیدی `class` استفاده می‌شود:

```php
<?php
class ماشین {
    public $رنگ;
    public $مدل;

    public function نمایش_مشخصات() {
        echo "این ماشین $this->رنگ و مدل $this->مدل است.";
    }
}
?>
```

### 6.1.2 ایجاد یک شیء

برای ایجاد یک شیء از یک کلاس، از کلمه کلیدی `new` استفاده می‌کنید:

```php
<?php
$ماشین_من = new ماشین();
$ماشین_من->رنگ = "قرمز";
$ماشین_من->مدل = "BMW";
$ماشین_من->نمایش_مشخصات(); // نمایش "این ماشین قرمز و مدل BMW است."
?>
```

## 6.2 سازنده‌ها و تخریب‌کننده‌ها

### 6.2.1 سازنده‌ها

یک سازنده (Constructor) متدی خاص است که هنگام ایجاد یک شیء جدید به طور خودکار فراخوانی می‌شود. در PHP، سازنده با نام `__construct()` تعریف می‌شود:

```php
<?php
class ماشین {
    public $رنگ;
    public $مدل;

    public function __construct($رنگ, $مدل) {
        $this->رنگ = $رنگ;
        $this->مدل = $مدل;
    }

    public function نمایش_مشخصات() {
        echo "این ماشین $this->رنگ و مدل $this->مدل است.";
    }
}

$ماشین_من = new ماشین("قرمز", "BMW");
$ماشین_من->نمایش_مشخصات(); // نمایش "این ماشین قرمز و مدل BMW است."
?>
```

### 6.2.2 تخریب‌کننده‌ها

یک تخریب‌کننده (Destructor) متدی خاص است که هنگام نابود شدن شیء فراخوانی می‌شود. در PHP، تخریب‌کننده با نام `__destruct()` تعریف می‌شود:

```php
<?php


class ماشین {
    public $رنگ;
    public $مدل;

    public function __construct($رنگ, $مدل) {
        $this->رنگ = $رنگ;
        $this->مدل = $مدل;
    }

    public function __destruct() {
        echo "شیء ماشین نابود شد.";
    }

    public function نمایش_مشخصات() {
        echo "این ماشین $this->رنگ و مدل $this->مدل است.";
    }
}

$ماشین_من = new ماشین("قرمز", "BMW");
$ماشین_من->نمایش_مشخصات(); 
// هنگام پایان اجرای اسکریپت، تخریب‌کننده فراخوانی می‌شود.
?>
```

## 6.3 وراثت (Inheritance)

وراثت یکی از اصول اصلی OOP است که به شما امکان می‌دهد یک کلاس جدید ایجاد کنید که ویژگی‌ها و متدهای یک کلاس دیگر را به ارث ببرد. برای استفاده از وراثت در PHP، از کلمه کلیدی `extends` استفاده می‌شود:

```php
<?php
class حیوان {
    public $نام;

    public function __construct($نام) {
        $this->نام = $نام;
    }

    public function صدا() {
        echo "$this->نام صدا می‌کند.";
    }
}

class سگ extends حیوان {
    public function صدا() {
        echo "$this->نام پارس می‌کند.";
    }
}

$حیوان_من = new سگ("راکس");
$حیوان_من->صدا(); // نمایش "راکس پارس می‌کند."
?>
```

## 6.4 اینترفیس‌ها و کلاس‌های انتزاعی

### 6.4.1 اینترفیس‌ها

اینترفیس‌ها به شما اجازه می‌دهند تا یک الگو برای کلاس‌ها تعریف کنید. کلاس‌هایی که یک اینترفیس را پیاده‌سازی می‌کنند، باید تمامی متدهای تعریف شده در اینترفیس را پیاده‌سازی کنند:

```php
<?php
interface پرنده {
    public function پرواز();
}

class عقاب implements پرنده {
    public function پرواز() {
        echo "عقاب پرواز می‌کند.";
    }
}

$پرنده_من = new عقاب();
$پرنده_من->پرواز(); // نمایش "عقاب پرواز می‌کند."
?>
```

### 6.4.2 کلاس‌های انتزاعی

کلاس‌های انتزاعی (Abstract Classes) به شما امکان می‌دهند تا متدهایی را تعریف کنید که باید در کلاس‌های فرزند پیاده‌سازی شوند. همچنین می‌توانید متدهای معمولی در کلاس انتزاعی تعریف کنید:

```php
<?php
abstract class شکل {
    abstract public function محیط();
    
    public function اطلاعات() {
        echo "این یک شکل است.";
    }
}

class دایره extends شکل {
    public $شعاع;

    public function __construct($شعاع) {
        $this->شعاع = $شعاع;
    }

    public function محیط() {
        return 2 * 3.14 * $this->شعاع;
    }
}

$شکل_من = new دایره(5);
echo $شکل_من->محیط(); // نمایش محیط دایره
?>
```

## 6.5 اصول SOLID در طراحی شیءگرا

SOLID مجموعه‌ای از اصول است که برای طراحی نرم‌افزارهای شیءگرا به کار می‌رود:

- **Single Responsibility Principle (SRP)**: هر کلاس باید فقط یک وظیفه خاص داشته باشد.
- **Open/Closed Principle (OCP)**: کلاس‌ها باید برای گسترش باز و برای تغییر بسته باشند.
- **Liskov Substitution Principle (LSP)**: کلاس‌های مشتق باید بتوانند جایگزین کلاس‌های والد خود شوند.
- **Interface Segregation Principle (ISP)**: استفاده از چندین اینترفیس خاص به جای یک اینترفیس عمومی.
- **Dependency Inversion Principle (DIP)**: ماژول‌های سطح بالا نباید به ماژول‌های سطح پایین وابسته باشند؛ هر دو باید به اینترفیس‌ها وابسته باشند.

## تمرین‌های فصل ششم

1. **ایجاد یک کلاس ساده**: یک کلاس به نام `ماشین` ایجاد کنید که دارای دو ویژگی `رنگ` و `مدل` باشد. یک متد به نام `نمایش_مشخصات` تعریف کنید که مشخصات ماشین را نمایش دهد.

2. **استفاده از سازنده**: یک سازنده برای کلاس `ماشین` ایجاد کنید که هنگام ایجاد شیء، `رنگ` و `مدل` را مقداردهی کند.

3. **وراثت**: یک کلاس به نام `حیوان` ایجاد کنید که یک ویژگی `نام` داشته باشد. سپس یک کلاس به نام `سگ` ایجاد کنید که از `حیوان` به ارث برده و متد `صدا` را برای سگ پیاده‌سازی کند.

4. **ایجاد اینترفیس**: یک اینترفیس به نام `پرنده` ایجاد کنید که متدی به نام `پرواز` داشته باشد. سپس یک کلاس به نام `عقاب` ایجاد کنید که اینترفیس `پرنده` را پیاده‌سازی کند.

5. **کلاس انتزاعی**: یک کلاس انتزاعی به نام `شکل` ایجاد کنید که متدی به نام `محیط` داشته باشد. سپس یک کلاس `دایره` ایجاد کنید که این کلاس انتزاعی را به ارث برده و متد `محیط` را پیاده‌سازی کند.
